# 44krei Personal Site Design

## 背景

目标是以 `https://ianneo.xyz` 为视觉和交互蓝本，做一个接近 1:1 美术气质的个人网站，但内容主体替换为邓又齐 / DH1Q / 44krei 本人的公开叙事、项目和实验。

当前工作目录为空目录，站点需要从零开始搭建。

## 设计目标

- 保留原站的视觉语言：窄内容宽度、等宽字体、白底浅边框、橙色强调色、轻量 hover、Hero 动态背景、微信弹窗。
- 页面结构尽量贴近原站：首页、关于页、详情页、联系弹窗、简洁页脚。
- 内容使用 Obsidian 中的公开导向资料，不使用账号备忘或任何敏感信息。
- 人设采用更锋利、更真实的版本：经历过低谷与重建，但表达克制，不卖惨、不说教。

## 非目标

- 不做传统花哨作品集。
- 不做后台、CMS、登录或复杂数据管理。
- 不复刻原站的具体文案、真实联系方式、外部账号地址。

## 站点结构

### 首页

- 顶部导航：头像位、首页、关于、联系按钮。
- Hero：动态网格背景、标签行、主标题、副标题、主 CTA。
- 项目区：使用与原站一致的 `list-row` 语法展示作品/项目/实验。
- 轨迹区：替代原站“成就”，展示真实经历、跨学科能力、重建路径。
- 联系区：邮件合作、微信联系、二维码按钮位。因无确认过的公开联系方式，首版允许使用占位文案。

### 关于页

- 返回入口。
- 头像和名字。
- 个人介绍。
- 我在做什么。
- 里程碑。
- 常见问题。

### 详情页

- 使用一个可复用模板，为每个重点项目生成单独页面。
- 详情结构与原站产品详情页同气质：项目定位、阶段、关键要点、下一步。

### 联系弹窗

- 全站复用。
- 保留类似原站的轻玻璃卡片与复制按钮体验。
- 联系信息在没有用户确认前使用可替换占位内容，并在页面中表达“资料整理中”。

## 视觉系统

### 风格关键词

- 极简
- 克制
- 有呼吸感
- 轻科技
- 手工精度
- 白底中的动势

### 视觉变量

- 页面最大宽度约 `768px`
- 主背景 `#ffffff`
- 主文字接近 `#0a0a0a`
- 次级文字接近 `#6b7280`
- 强调色接近 `#ff8c42`
- 边框接近 `#eaeaea`
- 主字体风格为等宽气质，优先 `JetBrains Mono`

### 动效

- Hero 动态网格背景，保留原站“张力流线”效果。
- 页面背景保留轻量漂浮光斑。
- 列表项 hover 有轻位移和底色浮现。
- CTA 保留扫光效果。
- 首屏与区块加载使用轻 reveal 动画。

## 内容映射

### 首页 Hero

- 标签行：`设计师 · 实验型创作者 · 概率决策者`
- 主标题：突出“重建自己”和“把想法做出来”
- 副标题：结合低谷、重建、设计、AI、实验、概率决策

使用可信能力证据替代夸张履历，重点强调：

- 从破产低谷到认知重建

## 技术设计

- 使用 Astro 构建，便于贴近原站技术路径和输出静态站点。
- 样式使用原生 CSS 与 CSS 变量，不引入 Tailwind，减少空项目下的搭建成本。
- 内容先使用本地 TypeScript 数据文件维护，避免过早引入 CMS。
- 动效脚本拆成独立组件，便于后续替换。

## 页面与文件边界

- `src/layouts/BaseLayout.astro` 负责全局头尾、SEO、背景层。
- `src/components/*` 负责导航、Hero、列表区、联系弹窗、详情模块、页脚等。
- `src/data/site.ts` 负责站点文案、项目、轨迹、FAQ、联系方式占位。
- `src/pages/index.astro`、`src/pages/about.astro`、`src/pages/projects/[slug].astro` 负责页面装配。
- `public/` 放置头像占位、二维码占位和 favicon。

## 风险与处理

- 缺少确认过的公开头像与联系方式：首版使用安全占位素材与替换提示。
- 原站动态背景较复杂：优先实现视觉近似和性能可接受的版本，不强追逐像素级一致。
- 当前目录不是 Git 仓库：文档可以保存，但无法按 skill 要求提交 commit。

## 交付标准

- 本地可运行。
- 首页、关于页、至少 3 个详情页可访问。
- 视觉气质明显接近参考站。
- 内容已替换为 44krei / DH1Q 的公开叙事。
- 构建通过。
